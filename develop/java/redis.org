* Redis缓存与数据库数据一致性解决方案
  操作策略：
  1. 先写数据库，再删除缓存：若前者成功，后者失败，则DB中是最新数
     据，缓存中是旧数据，数据不一致。
  2. 先删除缓存，再写数据库：若前者成功，后者失败，则缓存中无数据，DB
     中是旧数据。

  因此，第2种策略最佳。
